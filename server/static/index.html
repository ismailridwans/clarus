Three Escalating Tasks
Each task requires a different investigation strategy and reasoning depth
Deductive Liability
Easy
A patient receives a bill that doesn't match their EOB. The agent must gather records, identify the billing system error, and issue the correct refund.
Responsible party: billing_system_error
Resolution: refund
Grader checks: 17
Abductive Conflict
Medium
Conflicting signals between EOB, provider records, and NCCI rules. The agent must reason abductively to identify the insurer's wrongful denial and file a modifier exception appeal.
Responsible party: insurer_wrong
Resolution: appeal
Grader checks: 22
Adversarial Fabrication
Hard
The provider has submitted a fabricated Good Faith Estimate. After filing an NSA dispute, the agent faces 3 adversarial counter-arguments it must reject with cited evidence.
Responsible party: provider_fraud
Resolution: nsa_dispute
Grader checks: 28

API Reference
Standard OpenEnv interface — three endpoints
Method Path Description Body / Params
POST /reset Start a new episode. Returns initial observation. task_name, seed (optional)
POST /step Execute one action. Returns observation, reward, done, info. action.action_type, action.parameters
GET /state Return current episode state without advancing. —
GET /health Health check. Returns {"status":"ok"}. —
Example: Reset + First Step
# Reset a deductive_liability episode
POST /reset
{"task_name": "deductive_liability", "seed": 1101}
 
# First action — authenticate the patient
POST /step
{"action": {"action_type": "authenticate_patient", "parameters": {}}}
 
# Response
{"reward": 0.05, "done": false, "observation": {"step_number": 1, ...}}
